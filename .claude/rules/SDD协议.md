# SDD (Software Design Document) Protocol

## 文档结构

1. **概述** - 目的、范围、定义
2. **系统架构** - 组件图、模块关系
3. **接口定义** - API、消息格式
4. **数据结构** - 数据模型、数据库schema
5. **行为设计** - 状态机、流程图
6. **非功能性需求** - 性能、安全、可靠性

## 编写要求

- 明确标注版本号和日期
- 所有接口需提供示例
- 变更记录在文档末尾

## 审阅流程

1. 草稿编写 → 2. 同行评审 → 3. 架构师审核 → 4. 批准发布

## 文件命名

`SDD_<项目>_<版本>_<日期>.md`
